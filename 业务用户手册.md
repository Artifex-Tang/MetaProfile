## 1. 系统定位与工作流

MetaProfile 面向**产业技术情报**分析，核心流程如下：

```
原始资料                系统录入              分析产出
─────────              ──────────            ────────
论文/专利          →   技术画像    ─┐
项目报告           →   项目画像    ─┤→ 扫描监测 → 前沿技术榜单
机构公开资料       →   机构画像    ─┤→ 弱信号发现 → 新技术提示
人员简历/报告      →   人员画像    ─┘→ 选题服务 → 研究选题建议
```

**角色分工建议：**

| 角色 | 主要工作 |
|------|---------|
| 数据录入员 | 收集原始资料，按规范整理 JSON 导入 |
| 情报分析师 | 触发扫描、解读结果、提交选题反馈 |
| 课题负责人 | 评审选题、决策立项 |

## 2. 数据来源指引

### 技术画像数据来源

| 数据类型 | 推荐来源 |
|---------|---------|
| 学术论文 | 知网 / Web of Science / Semantic Scholar |
| 专利文献 | 智慧芽 / 国家知识产权局 / USPTO / EPO |
| 技术报告 | 各部委公开报告 / IEEE / MITRE |

| 数据类型 | 推荐来源 |

| 国际项目 | DARPA 项目数据库 / 欧盟 Horizon 项目库 |

| 企业 | 天眼查 / 企查查 / 年报 |
| 境外机构 | MIT / Stanford / Fraunhofer 等官网 |

### 人员画像数据来源

### 3.3 批量导入（多条）

将多个 JSON 对象放入数组，保存为 `.json` 文件：

```json
[
  { "tech_name_cn": "量子纠错技术", ... },
  { "tech_name_cn": "拓扑量子计算", ... },
  { "tech_name_cn": "量子互联网", ... }
]
```

**操作步骤：**
1. 打开系统 → 「技术画像」页面
2. 点击「批量导入」按钮
3. 选择准备好的 `.json` 文件
4. 等待提示「导入成功: X 条」

## 7. 扫描监测：触发与解读

### 7.1 何时触发扫描

建议在以下情况触发前沿技术扫描：

- **定期**：每月月初触发上月数据扫描
- **专项**：特定技术领域出现重大事件后
- **备选**：重大政策发布、大型学术会议结束后

### 7.2 操作步骤

1. 进入「扫描监测」页面
2. 选择**分析时间范围**（建议跨度 1~6 个月）
3. 选择**技术领域**（不选则扫描全部领域）
4. 点击「开始扫描」
5. 约 2~10 秒后列表自动刷新

### 7.3 结果字段解读

**前沿技术评分体系：**

| 字段 | 含义 | 判断标准 |
|------|------|---------|
| `fusion_score` | 融合评分（综合） | > 0.7 为高度关注 |
| `burst_score` | 爆发评分 | 近期发文/专利突增 |
| `patent_score` | 专利评分 | 专利申请密度 |
| `citation_score` | 引用评分 | 论文被引用量 |
| `invest_score` | 投资评分 | 风险投资 / 政府投入 |
| `policy_score` | 政策评分 | 政策文件提及密度 |
| `trl_level` | 技术成熟度 | 1~9级，7以上可工程应用 |

**状态说明：**
- `emerging`：新兴，初步出现苗头
- `confirmed`：确认，多维度信号验证
- `tracking`：跟踪中

**告警级别：**
- `critical`（红）：需立即关注，可能存在重大突破或差距
- `warn`（橙）：值得关注，趋势明显
- `info`（蓝）：一般提示

### 7.4 分析建议

融合评分 > 0.7 且 TRL ≥ 4 的技术，建议：
1. 点击「详情」查看各维度评分
2. 与已有技术画像比对
3. 若系统中无对应画像，补录技术画像
4. 将高分技术纳入下次选题生成

## 8. 新技术发现：弱信号分析

### 8.1 弱信号概念

弱信号指尚未形成主流认知、但在多个数据源中同时出现的技术苗头。典型特征：
- 关键词在不同领域文献中同时涌现
- 相关专利申请量开始抬头
- 多个独立研究团队同期发表

### 8.2 操作步骤

1. 进入「新技术发现」页面
2. 点击「触发发现扫描」（建议每季度一次）
3. 查看弱信号列表

### 8.3 字段解读

| 字段 | 含义 | 解读 |
|------|------|------|
| `keywords` | 信号关键词 | 代表该技术苗头的核心词 |
| `strength` | 信号强度 | 0~1，越高越显著 |
| `novelty` | 新颖度 | 0~1，越高越前沿 |
| `coherence` | 一致性 | 0~1，多源信号的一致程度 |
| `status` | 信号状态 | `active`/`confirmed`/`dismissed` |

### 8.4 关联网络图使用

点击「网络图」，进入 G6 可视化界面：

- **蓝色节点**：相关技术
- **绿色节点**：相关机构
- **橙色节点**：相关人员
- **紫色节点**：信号本体

操作：鼠标拖拽节点重排布局，滚轮缩放，点击节点查看 ID。

**分析价值**：
- 同一信号连接多个顶级机构 → 国际关注度高
- 信号连接多个不同领域技术 → 可能是交叉技术突破点
- 信号关联多个关键人员 → 核心团队已形成

## 9. 选题服务：生成与评审

### 9.1 生成前准备

系统生成选题质量与已录入数据正相关。建议在生成前确认：

- [ ] 技术画像库已覆盖目标领域主要技术
- [ ] 扫描监测已完成最近一期
- [ ] 弱信号库有新的发现记录

### 9.2 生成操作

1. 进入「选题服务」页面
2. 点击「生成选题」
3. 填写参数：
   - **生成数量**：建议 10~20 条，评审工作量适中
   - **分析周期**：填写希望参考的数据时间范围（可选）
4. 点击确认，约 3~30 秒后列表刷新

### 9.3 评分维度解读

每条选题包含以下评分（0~1）：

| 评分 | 含义 |
|------|------|
| `score_hot` | 热度评分：当前社会/学术关注程度 |
| `score_policy` | 政策评分：与国家政策方向的契合度 |
| `score_impact` | 影响力评分：潜在研究影响力 |
| `score_dedup` | 去重评分：与已有选题的差异度（越高越新） |
| `score_llm_gen` | LLM生成质量：AI认为该选题的价值 |
| `final_score` | 综合评分：加权融合 |
| `review_novelty` | 新颖度评审 |
| `review_importance` | 重要性评审 |
| `review_feasibility` | 可行性评审 |
| `review_expression` | 表述质量评审 |

### 9.4 评审工作流

**推荐评审流程：**

```
第一轮（快速筛选）：
  - 按 final_score 降序排列
  - 取前 50% 进入细读
  - 其余直接标记 reject

第二轮（细读评审）：
  - 逐条点击「详情」阅读摘要和评审依据
  - 填写反馈：
    · accept：直接立项
    · revise：修改方向后立项
    · reject：不适合当前方向
```

**提交反馈操作：**

1. 点击选题标题进入详情抽屉
2. 点击右上角「提交反馈」
3. 选择结论（接受 / 拒绝 / 修改）
4. 打分（1~10，10 分最优）
5. 填写评审意见（修改类必填，说明修改方向）
6. 点击确认

**反馈示例（修改类）：**
> 评分：7  
> 意见："选题方向正确，但建议聚焦国内自主可控角度，对比国外差距，提出针对性突破路径，而非泛泛综述。"

## 11. 数据质量规范

### 11.1 通用原则

- **来源标注**：`remark` 字段注明数据来源（文献编号 / 网址 / 报告名称）
- **时间精度**：能到日尽量到日，不确定时用月份第一天代替（如 `2023-03-01`）
- **置信度**：`confidence` 字段按以下标准填写：

| 来源质量 | confidence 建议值 |
|---------|------------------|
| 官方公开文件、顶刊论文 | 0.9~1.0 |
| 主流媒体报道、机构官网 | 0.7~0.9 |
| 二手引用、行业分析报告 | 0.5~0.7 |
| 推断或估计 | 0.3~0.5 |
| 未经核实的传言 | < 0.3（不建议录入）|

### 11.2 技术画像质量要求

- `tech_summary`：不少于 **100 字**，需包含技术原理和应用方向
- `current_status`：至少 **2 个观点**，观点间用"观点1：…观点2：…"格式分隔
- `trend`：至少 **2 个观点**，时间维度清晰（近期 / 中期 / 远期）
- `tech_domain`：粒度适中，不要过宽（如"信息技术"）也不要过窄（如"某专利编号"）

### 11.3 常见错误与修正

| 错误 | 修正 |
|------|------|
| `tech_domain: ["技术"]` | 改为具体领域：`["量子计算", "量子通信"]` |
| `current_status: "正在发展"` | 展开为具体观点："观点1：谷歌已实现...观点2：国内差距..."|
| `confidence: 1.0`（估计数据）| 估计数据不应超过 0.7 |
| `invention_date: "2023"` | 不符合日期格式，改为 `"2023-01-01"` |
| 项目 `name_cn` 写成字符串 | 必须为数组：`["项目名称"]` |

### 11.4 数据更新建议

| 画像类型 | 更新频率 |
|---------|---------|
| 技术画像 | 每季度检查热点领域，及时补录新技术 |
| 项目画像 | 重大项目立项/结项时更新 |
| 机构画像 | 半年更新一次规模、负责人等动态字段 |
| 人员画像 | 有新发表或职位变动时更新 |
