## 1 范围

### 1.1 标识

本大纲适用于 MetaProfile 产业技术情报系统 V1.0，规定系统功能测试、接口测试与单元测试的范围、方法、环境、进度与用例。

### 1.2 系统概述

系统由前端（React + nginx）与单一复合后端（FastAPI，端口 8000）组成，提供技术、项目、机构、人员四类画像的采集、检索与关联分析，以及扫描监测、新技术发现、选题服务、系统配置等功能。前端按 `/api-*` 前缀经 nginx 反向代理至复合后端。

### 1.3 文档概述

本大纲第 2 章列出引用文件；第 3 章给出测试概述；第 4 章给出两轮测试进度；第 5 章列出测试项；第 6 章给出测试用例（R1 基线与 R2 新增）；第 7 章为附录。

## 3 测试概述

### 3.1 测试范围

覆盖前端九个功能页面（总览、技术/项目/机构/人员画像、扫描监测、新技术发现、选题服务、系统配置）的全部可达交互，复合后端接口，以及前端单元逻辑。

### 3.2 测试方法

- **功能测试**：按页面交互逐项验证（搜索、导入、详情、图谱、扫描、生成、反馈、配置）。
- **接口测试**：通过 Swagger 与 curl 验证复合后端 `/api/v1` 路由。
- **单元测试**：Vitest + Testing Library 对可隔离的纯函数、钩子与组件进行自动化测试。
- **回归测试**：R2 执行 R1 全部用例，确认新功能未引入回归。

### 3.3 测试类型

功能测试、接口测试、单元测试、回归测试。
