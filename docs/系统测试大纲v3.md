### 文档概述

当前全球科技竞争日趋激烈，及时、准确地掌握前沿技术布局、关键人才与机构动向，对产业技术决策至关重要。然而，技术情报散落于专利、文献、项目、机构公开信息等多源异构数据中，存在采集难、关联弱、研判慢等挑战。MetaProfile 产业技术情报系统面向上述需求，构建"采集—画像—发现—研判—选题"的情报闭环，对技术、项目、机构、人员四类实体建立结构化画像，并提供前沿扫描、弱信号发现与选题辅助决策能力。

本大纲规定 MetaProfile 产业技术情报系统 V1.0 的测试范围、测试内容与方法、测试环境、实施计划与质量评价，作为系统测试与质量评价的依据。

### 软件概述

MetaProfile 由前端与单一复合后端组成，采用 Docker Compose 编排，默认启动全部服务。

- 复合后端 `backend`（端口 8000）：单一 FastAPI 应用，聚合画像层（技术 / 项目 / 机构 / 人员）、分析层（扫描监测 / 新技术发现 / 选题服务）与配置层的全部路由，共 77 条 `/api/v1` 路由。
- 前端 `frontend`（端口 80）：React 单页应用，nginx 监听 80，按前缀 `/api-tech`、`/api-project`、`/api-org`、`/api-person`、`/api-scan`、`/api-discovery`、`/api-topic`、`/api-settings` 反向代理至 `backend:8000`，前缀剥离后落到后端 `/api/v1`。
- 基础设施：PostgreSQL（5432，主库）、Elasticsearch（9200，全文检索）、Redis（6379，缓存）。
- 增强组件：Neo4j（7474/7687，实体关系图谱）、RabbitMQ（5672/15672，异步任务队列）、LiteLLM（4000，大模型代理）。
- 一次性任务：`migrate`（数据库迁移）、`seed`（示例数据灌入）。

系统九个功能页面：总览、技术画像、项目画像、机构画像、人员画像、扫描监测、新技术发现、选题服务、系统配置。

## 测试内容与方法

### 测试范围

覆盖前端九个功能页面的全部可达交互、关系图谱跨画像跳转、复合后端接口、前端单元逻辑，以及软件基础性能与易用性。本次测试分两轮：第一轮（R1）九模块基线功能共 69 项用例；第二轮（R2）跨画像跳转 + 前端单元 + R1 回归共 10 项用例；合计 79 项。

| 质量特性 | 子特性 | 测试重点 |
|---|---|---|
| 功能性 | 完整性、正确性、适合性、准确性 | 九页面功能、跨画像跳转、接口契约 |
| 性能效率 | 时间特性、资源利用性 | 页面加载、检索、图谱渲染、异步任务响应 |
| 易用性 | 可辨识性、易学性、易操作性、用户差错防御 | 交互一致性、空态/错误提示、边界防护 |

### 子特性的测试要求

#### 功能性测试

按页面与功能模块逐项验证用户可达交互，每个功能用例含编号、测试项、前置条件、测试步骤、预期结果。功能用例经前端交互触发，背后接口与反代正确性随之验证。

#### 性能测试

在测试环境下选取典型操作观测响应时间，给出预期与实测对比。

### 测试策略与方法

- 功能测试：按页面交互逐项验证（黑盒，等价类与边界值结合）。
- 接口测试：经前端交互透传验证，辅以 Swagger 与 curl 核对复合后端 `/api/v1` 路由。
- 单元测试：Vitest + Testing Library 对可隔离的纯函数、钩子与组件自动化测试。
- 回归测试：R2 重跑 R1 全部用例，确认新功能未引入回归。
- 性能测试：典型操作计时观测。

### 测试充分性要求与有效性分析

- 充分性：九页面全部可达交互均有对应用例；跨画像跳转覆盖可跳/不可跳/链式/刷新/分享全部分支；单元测试覆盖跳转相关纯函数、钩子、组件。
- 有效性：用例预期结果明确可判，执行结果与预期逐一比对；缺陷按等级登记并回归关闭。
